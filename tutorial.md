# Claude Code Inspector 使用教程

## 这是什么

Claude Code Inspector 是一个检查工具。它会扫描你电脑上 Claude Code 的所有配置，然后生成一份网页报告，告诉你：

- Claude 现在能做什么、不能做什么
- 哪些配置有问题、怎么修
- 你的配置能打多少分

就像体检报告一样，只不过体检的对象是你的 Claude Code。

---

## 怎么用

只需要两步：

**第一步：把文件放到电脑上**

把收到的 `generate-dashboard.js` 文件保存到电脑上任意位置，记住放在哪了。

**第二步：在 Claude Code 里运行**

打开 Claude Code，输入下面这句话：

```
帮我运行一下 node /文件存放的路径/generate-dashboard.js
```

比如你把文件放在了桌面上，就输入：

```
帮我运行一下 node ~/Desktop/generate-dashboard.js
```

运行完成后，当前文件夹里会出现一个 `claude-code-inspector.html` 文件。双击打开它，报告就出来了。

---

## 报告里有什么

### 1. 配置得分

页面最上面有一个分数（满分 100）和等级（S/A/B/C/D）。分数越高，说明你的 Claude Code 配置越完善。

### 2. 能力地图

这里用卡片展示 Claude 目前具备的能力，比如：

- 能不能搜索网页
- 能不能操作浏览器
- 有没有记住你的习惯

绿色图标 = 已开启，灰色 = 还没配置。

### 3. 待优化项

这是最实用的部分。每一条都是一个可以改进的地方，展开后能看到：

- 这个问题是什么意思
- 怎么修
- 一个「复制修复命令」按钮 — 点一下，去 Claude Code 里粘贴，就能修好

---

## 怎么用它管理自己的项目

每次你开始一个新项目，或者觉得 Claude 不够好用的时候，可以按这个流程操作：

**1. 先跑一次检查**

生成报告，看看分数和待优化项。

**2. 从上到下修复**

报告里的问题已经按重要程度排好了。从第一条开始，点「复制修复命令」，粘贴到 Claude Code 里执行。修一条算一条。

**3. 再跑一次，看分数变化**

修完几条之后再运行一次，看看分数是不是上去了。

**4. 定期复查**

就像定期体检一样，每隔一段时间跑一次，保持 Claude Code 在最佳状态。

## 常见问题

**Q：运行后报错了怎么办？**

确认你的电脑上装了 Node.js。在 Claude Code 里输入「帮我检查一下有没有安装 Node.js」，它会帮你确认。

**Q：报告里的信息会泄露吗？**

不会。所有 API 密钥和密码在报告里都会自动打码。生成的 HTML 文件可以安全分享。

**Q：每次都要重新运行吗？**

是的。每次运行会重新扫描你当前的配置，生成最新的报告。
